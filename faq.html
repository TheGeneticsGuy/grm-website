Troubleshooting & Syncing

This is a known anomaly. When you first join a guild, the "Addon-to-Addon" communications that transmit over the GUILD channel are not activated until you fully exit the game and log back in.

Simply logging to the character select screen or reloading UI (/reload) is not enough. You must fully restart the WoW client. This issue, which appears to have started around Patch 11.0, prevents GRM from determining who fellow addon users are until a client restart occurs.

GRM often gets the blame, but this is caused by certain DKP addons (notably Bohemian DKP) automatically placing `0100` in player notes. Since GRM syncs note changes, it propagates this change.

If you need to fix this, and you have a local backup or restore point within GRM, you can use the following script to restore notes. Use cautiously as it overwrites all public notes guild-wide:

/run GRM_API.RestorAllPublicNotesFromSave()

The addon author does not regularly update the skin file for GRM. You can manually install the latest working file here: GitHub: GRM AddOnSkins File.

Download the Guild_Roster_Manager.lua file and copy it to:

\Interface\AddOns\AddOnSkins\Skins\AddOns\

If AddOnSkins has blacklisted GRM due to errors, run this script and then re-enable GRM:

/run AddOnSkinsDS={};ReloadUI()

Data & Backups

The server does not provide this information directly, but there are a few workarounds:

1. Check WoWProgress

If your guild is tracked on wowprogress.com, you can often find rough join dates there.

2. Use the In-Game Command (Retail Only)

If players joined after the 2018 (BFA 8.0) Communities update, the server has their join date logged. Players can run this macro themselves and report the date to you:

/run local t=date("*t",math.floor((tonumber(string.format("%.0f",C_Club.GetClubInfo(C_Club.GetGuildClubId()).joinTime))/1000000)+0.5));print("Date Joined: "..t.year.."-"..t.month.."-"..t.day);

3. Mass Set Join Date (Last Resort)

If you simply want to fill the blanks with a placeholder date (e.g., Aug 25, 2023) for everyone missing a date:

/run GRM_API.SetAllUnknownJoinDates(25,8,2023)

Your data is stored in your WoW Directory here:

_retail_\WTF\Account\YOUR_ACCOUNT\SavedVariables\Guild_Roster_Manager.lua
  • To Backup: Log out of the game, copy that `.lua` file, and paste it somewhere safe.
  • To Restore: Log out, delete the current file in that folder, and paste your backup copy in its place.

Features & Limitations

GRM Logs are personal to you and are not synced. They are generated based on your own configuration settings and what you witnessed while online.

If you play at different times than other officers, your logs will differ. The log is only current from the day you installed the addon.

Custom GRM Notes: Yes. GRM logs who created or edited these notes.

Public/Officer Notes: No. Blizzard does not provide API access to see who modified standard in-game notes.

No. GRM relies on the modern Retail and Classic APIs. It will not function on private servers running older, modified API builds.
